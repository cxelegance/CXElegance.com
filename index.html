Elegance:

Marzio needed a website to display his cartoons, without the clutter usually found on blogs (archive lists, tags and categories), and without the usual weblog format for displaying posts.
Marzio was in Canada, I was in Buenos Aires; we communicated almost entirely via email - light, fun, positive emails. I built up a small requirements document for him to comment on and approve; he was busy working full-time during this project. I emailed him examples and prototypes and he asked for modifications when necessary.

Marzio was anxious to get content loaded; I formatted and entered 30 cartoons for him while I tested my "add cartoon" functionality. Our joint effort was very successful and enlightening!

I also gave Marzio some training via web conference so that he felt confident managing his content. I was flexible across time differences so as to suit his narrow availability. Long distance can be a total success!

A blog just didn't seem like the right solution. So I made one, using simple things like XML. I created simple classes for cartoons and cartoon lists, each of these classes were able to load/save data via PHP DOM XML, and they held and operated on their data in simple arrays. With XML, the RSS feed was super easy.

Marzio didn't need fancy in the back-end; I made very simple webpages for adding, modifying, deleting content. And I bullet-proofed the system using JavaScript form/data checking before sending to PHP.

Also, I created some fun ways to navigate through the cartoon collection, while keeping the interface simple and uncluttered.

I just wanted a decent website that shows what I do - and what I can do - while offering a neat-o user experience.
Well, I can say that I was a pleasure to work with. I wanted things to be grand but I let myself talk me into something simple yet cool.

I went on many great walks to the plaza near where I was staying in Buenos Aires, and during those walks beautiful ideas and solutions came to me. Then I'd get to work for a few hours, only to stop again and walk to get some delicious facturas (pastries)!

This is not the first page on the Internet to do this - but all the content is in one html file... how cool! I used CSS to organize the data, and jQuery picks out the data as required, while animating the transitions to different sections. If you took away all the jQuery and CSS, you'd see all the information in a simple, long page, and Google can index sections/links via hash tags.

Oh, and this site displays reasonably well on all sizes, from mobile to big screen, with very little trickery - essentially just letting the browser wrap content to the next line when things get narrow!
